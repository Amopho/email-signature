Some Notes

  • Inline css or Style tags are used in emails. You're allowed to link to external resources, but email clients/spam filters hate that, so separate css files are not used. This preserves the static nature of emails-- an email can't change from one day to the next.
  • An email sig is thought of as a small, static element. So an html table is used. In this case, I used a row with 3 columns.
  • The center row is optional, since the orange border could have been applied to the last column with some extra padding-left to achieve the same affect. But I enjoy the extra degree of freedom at minimal cost.
  • The graphics tend to be bigger than the display size. This creates better appearance on uhd monitors.
  • Links were added. The standard text-decoration doesn't match the design style, so I opted for icons to indicate clickability. On that note, the phone number is clickable as a convenience for call-able devices.
